# 六色电子纸每日同图多版本对照

本项目的 7.3 英寸相框是离散六色显示：黑、白、红、黄、蓝、绿。照片无法保留连续 RGB 色彩，需要先映射到面板实际可显示的色域，再用抖动把亮度与色彩层次分布到周围像素。

## 候选方案

对照保留差异足够明显、适合照片的四种完整管线，避免让相近 kernel 造成无效比较：

| 代号 | 实现 | 主要观察点 |
| --- | --- | --- |
| 平衡照片 | Sierra 扩散 | 颗粒、层次、速度的稳定基线 |
| 自然色调 | Tone compression + Lab/ΔE00 | 人像肤色、暗部和中间调 |
| 纸白高光 | Paper-white compensation + Lab/ΔE00 | 天空、云、白墙和高光细节 |
| 细节蓝噪声 | Blue-noise ordered dither + Lab/ΔE00 | 纹理、边缘、是否有规则条纹 |

所有候选都使用相同的六色硬件调色板和 `480x800` 裁剪；四张卡片使用同一日的源照片，差异只来自照片渲染管线。

## 如何对照

1. 在控制台打开“发布”。
2. 勾选“显示每日多版本对照工具”，然后点击“保存 Daily Dither”。这只打开管理台工具，不会改变设备当前显示或设备日图策略。
3. 点击“生成今日四版本”。控制台按需渲染今天同一张源图的四个候选，并以四宫格并排展示。
4. 在接下来几天重复生成并人工比较。建议记录：人像、天空/高光、植物绿色、暗部、颗粒感、纹理和整体偏好。
5. 认为某个方案最合适时，在该卡片点击“选为默认”。系统把对应算法和色表写为后续日图默认方案；对照工具仍可保留，方便继续验证。

## 设备行为、能耗与回滚

- 无论对照工具是否开启，相框设备始终请求并显示保存的默认日图方案；不会按日期轮换候选，也不会额外唤醒或增加设备拉图请求。
- 四版本生成只在管理台手动点击时发生，结果走服务端预览接口，不会改变已缓存的设备静态资源或插播任务。
- 若不再需要比较，关闭“显示每日多版本对照工具”并保存即可隐藏工具；当前保存的默认算法与色表不变。
- 插播仍使用创建任务时明确选择的渲染算法。
