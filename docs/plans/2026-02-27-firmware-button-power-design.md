# 固件按键语义与省电策略：KEY 手动同步 + 深睡定时 64-bit（2026-02-27）

## 目标

1. **KEY = 手动同步键**：短按唤醒后执行一次完整同步（联网→拉图→上报→深睡），不再默认打开 120 秒配置窗口，避免固定成本放大耗电。
2. **深睡定时使用 64-bit 计算**：避免 `seconds * 1000000` 这类计算在 32-bit 上溢出导致“睡眠时长异常/频繁唤醒”的隐蔽耗电问题。
3. **保留可靠的配网/配置入口（同时避免误触）**：
   - 长按 BOOT 作为“清 Wi-Fi 逃生口”
   - 长按 KEY 作为“非破坏性的本地配置入口”

---

## 背景与动机

历史现象表明：任何“每轮唤醒必做”的动作都会在长期运行中被无限放大，最终成为主要耗电项。此前固件里 `KEY` 唤醒会打开 120 秒 STA 配置窗口，这在误唤醒或误判按键时会形成灾难性的固定成本。

因此，本次调整优先做两件事：

1) **把 120 秒窗口从默认路径移出**（只在用户显式长按时进入）  
2) **把“按一下 KEY 想触发同步”做到尽量可靠**，同时不牺牲“误唤醒不联网”的省电保护

---

## 方案与取舍

### 方案 A（采用）：短按 KEY 手动同步；长按 KEY 打开 STA 配置窗口；长按 BOOT 清 Wi-Fi

**按键语义**

- `KEY` 短按：手动触发一次同步与上报（正常拉图逻辑，允许命中 `304`）
- `BOOT` 短按：强制刷新（绕过条件 GET，确保拿到正文并重渲染）
- `KEY` 长按（3 秒）：打开 STA 本地配置窗口（120 秒，不清 Wi-Fi）
- `BOOT` 长按（3 秒）：清 Wi-Fi 并进入 AP 配网模式（逃生口）

**省电要点**

- KEY 不再进入 120 秒窗口，避免“固定成本”
- 保留 `SPURIOUS_EXT1` 兜底：若未观察到按钮被按下，则直接跳过联网并快速进入 timer-only 深睡

**可用性要点**

- 唤醒原因判定增加“短采样窗口”：在几十毫秒内只要捕获到一次按下，就视为人为短按，避免“按了不生效”

### 方案 B：仍在 KEY 短按时打开较短（如 10-30 秒）的窗口

优点：改配置更方便。  
缺点：固定成本仍存在；在误唤醒场景下仍可能放大耗电。与“省电第一优先级”冲突，否决。

---

## 深睡定时：64-bit 计算约束

深睡 API 使用微秒计数，常见写法是 `seconds * 1000000`。如果 `seconds` 为 32-bit（或中间计算发生 32-bit 提升），长睡眠可能溢出，表现为：

- 实际睡眠远短于预期 → 频繁唤醒 → 耗电激增

本次约束为：

- 统一封装 `SecondsToMicroseconds(uint64_t seconds)`，确保乘法在 64-bit 下进行

---

## 验证方法（硬件在手时）

1. **短按 KEY**：串口应出现 `wake source=KEY, manual sync triggered`，并继续执行联网/拉图/上报流程。
2. **误唤醒**：串口应出现 `wake source=SPURIOUS_EXT1, skip network cycle`，设备进入 timer-only 深睡。
3. **长按 KEY 3 秒**：连接 Wi-Fi 后应打印 STA 配置窗口 URL（并在窗口到期后继续正常流程）。
4. **长按 BOOT 3 秒**：应清 Wi-Fi 并进入 AP 配网模式（`PhotoFrame-Setup`）。
