# 电池曲线增加散点（折线 + 散点叠加）设计

日期：2026-03-03  
范围：photoframe-orchestrator Web 控制台（电池曲线）

## 背景 / 问题

当前电池曲线使用折线绘制 `battery_percent` 与 `battery_mv`。当采样点很少（尤其只有 1 个点）时：

- 折线只有 `moveTo` 没有 `lineTo`，几乎不可见；
- 用户无法快速判断“设备到底有没有 checkin 上报采样点”。

这会导致验证电量上报、排查功耗问题时效率很低。

## 目标

- 让“是否有上报点”一眼可见（尤其是 1 个点的情况）。
- 保持现有折线趋势展示能力。
- 不引入新的后端接口与存储。
- 性能可接受：最多 ~2 万点（`/api/v1/power-samples` limit=20000）绘制不明显卡顿。

## 非目标

- 不做交互式 tooltip/hover 详情（后续可选增强）。
- 不改变采样口径（采样点只来自设备 checkin；不允许 UI 伪造上报点）。
- 不引入新的可视化库（继续使用现有 Canvas 绘制）。

## 方案（拍板）

采用“折线 + 散点叠加”：

1) 保留现有两条折线：
   - 电量（%）蓝线
   - 电压（mV）橙线（当存在有效 mv 范围时）

2) 对每个有效采样点叠加散点：
   - 使用 `fillRect` 画小方点（比 `arc` 更省 CPU）
   - 电量与电压点分别使用与折线一致的主色，并略降低透明度，避免遮挡

3) 高亮“最后一个有效点”：
   - 画更大的方点并加轻微描边
   - 便于快速确认“最近一次上报时间点”

## 可回滚性

- 变更集中在 `drawPowerChart()` 的前端绘制逻辑，不影响任何后端数据。
- 如出现视觉拥挤或性能问题，可单提交回滚散点绘制代码。
