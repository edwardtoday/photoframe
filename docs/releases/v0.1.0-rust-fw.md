# 发布说明：`v0.1.0-rust-fw`

- 发布日期：`2026-03-09`
- Tag：`v0.1.0-rust-fw`
- 对应提交：`703521c`
- 发布类型：里程碑（Rust 固件重写第一阶段）

## 本次发布目标

将相框固件主链从 C++ 迁移到 Rust，并打通“可编译 → 可刷机 → 真机联网拉图 → 周期休眠”的完整闭环，同时保持 orchestrator 配置/管控能力。

## 核心变更

1. Rust 固件主链打通（`firmware/photoframe-rs/`）
   - NVS 配置与设备身份
   - 多 Wi‑Fi 轮询连接
   - SNTP 校时
   - `device/config` / `device/next` / `device/checkin` 联调
   - 图片下载、BMP/JPEG 渲染、深睡决策
2. 图片链路增强
   - 同源图片请求自动携带 `X-PhotoFrame-Token`
   - orchestrator 新增内部预览端点：
     - `/api/v1/preview/current.bmp`
     - `/api/v1/preview/current.jpg`
   - 在无 `PUBLIC_DAILY_BMP_TOKEN` 时，`/api/v1/device/next` 优先下发内部 preview URL
3. 显示与稳定性修复
   - 开启 PSRAM 配置，修复渲染阶段内存分配失败
   - 支持并验证 `display_rotation=2`（180° 旋转）
4. 设备联网可运维性增强
   - 设备内置并确保以下 Wi‑Fi 资料存在：
     - `OpenWrt / sansiAX3`
     - `Qing-IoT / jiajuzhuanyong`
     - `Qing-AP / 64139772`
   - orchestrator 支持通过 `wifi_profiles` 对设备 Wi‑Fi 列表增删改查（完整替换语义）

## 验收结果（已完成）

- 构建：Rust 固件可通过 Docker 工具链稳定构建并导出刷机镜像
- 刷机：设备可刷写并启动 Rust 固件
- 联网：可在目标环境命中任意一个已知 Wi‑Fi 并获取 IP
- 联调：`device/config`、`device/next`、`device/checkin` 均成功返回 200
- 拉图：设备可成功拉取并渲染下发图片（含 JPEG/BMP）
- 休眠：周期结束进入预期休眠策略
- 旋转：`display_rotation=2` 生效，实机方向正确

## 使用方式

1. 构建固件：
   - `scripts/build-photoframe-rs.sh`
2. 刷机（整包）：
   - `~/.espressif/python_env/idf5.0_py3.13_env/bin/esptool.py --chip esp32s3 --port /dev/cu.usbmodem111201 --baud 115200 write_flash -z 0x0 firmware/photoframe-rs/dist/photoframe-rs.bin`
3. 串口观察：
   - `scripts/monitor-host.sh --once /dev/cu.usbmodem111201 115200`
